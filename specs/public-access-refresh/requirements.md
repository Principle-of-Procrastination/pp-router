# Requirements

1. The Web app and API must work without an application access key or Bearer session.
2. Public model calls must retain per-client and global rate limits, concurrency limits, request bounds, exact CORS, and generic upstream errors.
3. Public history responses must not expose stored user query text.
4. Chinese difficulty routing must recognize structural complexity including constraints, deliverables, scale, diagnostics, and multi-criteria decisions.
5. Short referential follow-ups must inherit the previous user task without including assistant text.
6. LiteLLM must use the latest stable release that passes the project's routing and streaming regressions; pre-releases are excluded.
7. CloudBase backend and frontend deployments must be smoke-tested after release.
